# Frontend i18n (中 / EN) — Design Spec

- **Date:** 2026-04-21
- **Status:** Design approved, ready for implementation plan
- **Scope:** Web UI (`web/src/`). **Zero backend runtime changes.** One test-only Go file is added (`config/descriptor/fixture_gen_test.go`, build-tagged) to generate a CI fixture; it does not affect production code paths.
- **Delivery:** Single PR

## 1. Goals

Ship a bilingual (English + Simplified Chinese) web UI for hermind, with architecture that supports adding further languages by dropping in one JSON file per locale. Coverage includes **both** frontend-owned hardcoded strings (`~57` occurrences across ~20 component files) and backend-delivered descriptor metadata (`Label` / `Summary` / `Help` / `GroupID` on every `config/descriptor/*.go` section and field, ~140 keys), because descriptor text is **~70%** of user-visible copy and English-only descriptors would make the Chinese UI feel broken.

### 1.1 Non-goals

- Backend `error.Error()` strings (REST handler responses) remain English. No `Accept-Language` handling server-side. No `go-i18n`, no `golang.org/x/text/message`.
- Date / number / currency formatting (UI does not display any).
- RTL languages.
- Lazy-loaded translation bundles (2 locales × ~200 keys ≈ 10 kB uncompressed; embed in main bundle).
- Crowdin / external translation workflow. Contribute via PR on JSON.
- Chinese web font loading (use OS `system-ui` / `PingFang SC` / `Microsoft YaHei`).
- URL query override (`?lang=zh-CN`). `localStorage` is enough.
- Lint rule auto-detecting untranslated JSX English literals. YAGNI.

## 2. Architecture

```
┌───────────────────────────────────────────────┐
│ Go backend (api/handlers_config.go etc.)      │
│ - Unchanged: /api/config/schema serves English│
│ - Does NOT read Accept-Language               │
│ - descriptor.Label/Help/Summary stays English │
└───────────────────────────────────────────────┘
                    │  JSON (English source-of-truth)
                    ▼
┌───────────────────────────────────────────────┐
│ React frontend (web/src/)                     │
│                                               │
│ ┌─────────────────────────────────────────┐   │
│ │ i18n runtime (react-i18next)            │   │
│ │  - init before React mounts             │   │
│ │  - detect navigator.language            │   │
│ │  - localStorage persistence             │   │
│ │  - <I18nextProvider> wraps <App />      │   │
│ └─────────────────────────────────────────┘   │
│                                               │
│ ┌──────────────┐  ┌────────────────────┐      │
│ │useTranslation│  │ useDescriptorT     │      │
│ │  (ui NS)     │  │   (descriptors NS) │      │
│ │              │  │                    │      │
│ │ Hardcoded    │  │ Overlays English   │      │
│ │ frontend     │  │ schema by dotted   │      │
│ │ strings      │  │ key; falls back to │      │
│ │              │  │ backend literal    │      │
│ └──────┬───────┘  └──────────┬─────────┘      │
│        │                     │                │
│        ▼                     ▼                │
│  locales/{en, zh-CN}/ui.json                  │
│  locales/{en, zh-CN}/descriptors.json         │
└───────────────────────────────────────────────┘
```

**Principles:**
- Backend zero-change (API contract stable; single source of truth for descriptors stays in Go).
- Two independent i18next namespaces: `ui` (frontend literals) vs `descriptors` (schema overlay).
- Descriptor lookup **must** pass the backend English literal as `fallback`. A missed key renders English, never a raw key string. This invariant is enforced at the TypeScript signature level.

## 4. i18n Runtime

### 4.1 Initialization (`web/src/i18n/index.ts`)

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import enUI from '../locales/en/ui.json';
import enDesc from '../locales/en/descriptors.json';
import zhUI from '../locales/zh-CN/ui.json';
import zhDesc from '../locales/zh-CN/descriptors.json';
import { detectInitialLanguage, SUPPORTED_LANGUAGES, STORAGE_KEY } from './detect';

export function initI18n(): Promise<unknown> {
  return i18n.use(initReactI18next).init({
    resources: {
      en:      { ui: enUI, descriptors: enDesc },
      'zh-CN': { ui: zhUI, descriptors: zhDesc },
    },
    lng: detectInitialLanguage(),
    fallbackLng: 'en',
    supportedLngs: [...SUPPORTED_LANGUAGES],
    defaultNS: 'ui',
    ns: ['ui', 'descriptors'],
    // CRITICAL: keySeparator disabled so descriptor field names like
    // "browserbase.api_key" can appear inside lookup keys without being
    // mis-parsed as nested paths. JSON resources use flat top-level keys
    // (see §5.1 and §7.3). pluralSeparator remains the default '_'.
    keySeparator: false,
    nsSeparator: ':',
    interpolation: { escapeValue: false },
    returnNull: false,
  }).then(() => {
    document.documentElement.lang = i18n.language;
  });
}

export function setLanguage(lang: string): void {
  i18n.changeLanguage(lang);
  localStorage.setItem(STORAGE_KEY, lang);
  document.documentElement.lang = lang;
}
```

### 4.2 Detection (`web/src/i18n/detect.ts`)

```typescript
export const SUPPORTED_LANGUAGES = ['en', 'zh-CN'] as const;
export type SupportedLanguage = (typeof SUPPORTED_LANGUAGES)[number];
export const STORAGE_KEY = 'hermind.lang';

export function detectInitialLanguage(): SupportedLanguage {
  // Priority 1: user's previous manual choice
  try {
    const stored = localStorage.getItem(STORAGE_KEY);
    if (stored && (SUPPORTED_LANGUAGES as readonly string[]).includes(stored)) {
      return stored as SupportedLanguage;
    }
  } catch { /* SSR / private-browsing safe */ }

  // Priority 2: browser preference
  const nav = typeof navigator !== 'undefined' ? (navigator.language || '') : '';
  if (nav.toLowerCase().startsWith('zh')) return 'zh-CN';

  // Priority 3: fallback English
  return 'en';
}
```

### 4.3 Mount Point (`web/src/main.tsx`)

```typescript
import { initI18n } from './i18n';

initI18n()
  .catch((err) => console.error('i18n init failed:', err))
  .finally(() => {
    ReactDOM.createRoot(document.getElementById('root')!).render(
      <StrictMode>
        <App />
      </StrictMode>
    );
  });
```

`.catch` logs init failure, `.finally` always renders. If init rejects the UI still mounts — i18next's `t()` then returns the `defaultValue` / key string, so users see English fallback rather than a blank screen. Init failure must never leave the UI blank.

## 5. Descriptor Overlay Contract

### 5.1 JSON Shape

**Flat top-level keys** (because `keySeparator: false` — see §4.1 — so nesting is not traversed and dotted field names stay intact). Example `web/src/locales/zh-CN/descriptors.json`:

```json
{
  "browser.label":                                    "浏览器",
  "browser.summary":                                  "浏览器自动化 provider。留空表示不启用浏览器集成。",
  "browser.fields.provider.label":                    "Provider",
  "browser.fields.provider.help":                     "浏览器自动化后端。留空则禁用。",
  "browser.fields.provider.enum.browserbase":         "Browserbase（云端）",
  "browser.fields.provider.enum.camofox":             "Camofox（本地）",
  "browser.fields.browserbase.api_key.label":         "Browserbase API 密钥",
  "browser.fields.browserbase.api_key.help":          "环境变量 BROWSERBASE_API_KEY 会在运行时覆盖此值。",
  "browser.fields.browserbase.project_id.label":      "Browserbase 项目 ID",
  "browser.fields.browserbase.project_id.help":       "环境变量 BROWSERBASE_PROJECT_ID 会在运行时覆盖此值。",

  "groups.runtime":  "运行时",
  "groups.models":   "模型",
  "groups.advanced": "高级"
}
```

Enum values use the dotted path `<section>.fields.<field>.enum.<value>`. Empty-string enum values (convention: "unset") are NOT translated — UI renders them via the `ui:field.none` key instead.

**Authoring ergonomics:** keys are sorted alphabetically / grouped by section prefix in each file; contributors `grep` by prefix to find all fields of a section. A Prettier / JSON formatter preserves insertion order, so editors can keep logical grouping by section.

### 5.2 Hook (`web/src/i18n/useDescriptorT.ts`)

```typescript
import { useTranslation } from 'react-i18next';

export function useDescriptorT() {
  const { t } = useTranslation('descriptors');

  return {
    sectionLabel(sectionKey: string, fallback: string): string {
      return t(`${sectionKey}.label`, { defaultValue: fallback });
    },
    sectionSummary(sectionKey: string, fallback: string): string {
      return t(`${sectionKey}.summary`, { defaultValue: fallback });
    },
    fieldLabel(sectionKey: string, fieldName: string, fallback: string): string {
      return t(`${sectionKey}.fields.${fieldName}.label`, { defaultValue: fallback });
    },
    fieldHelp(sectionKey: string, fieldName: string, fallback: string): string {
      return t(`${sectionKey}.fields.${fieldName}.help`, { defaultValue: fallback });
    },
    enumValue(sectionKey: string, fieldName: string, value: string, fallback: string): string {
      return t(`${sectionKey}.fields.${fieldName}.enum.${value}`, { defaultValue: fallback });
    },
    groupLabel(groupId: string, fallback: string): string {
      return t(`groups.${groupId}`, { defaultValue: fallback });
    },
  };
}
```

### 5.3 Invariant

**Every call site must pass `fallback` = backend English literal.** This is enforced by TypeScript: all signatures declare `fallback: string` as a required parameter. A call with `fallback` === `''` is a caller bug, not an i18n bug.

### 5.4 Call-Site Migration (illustrative)

```typescript
// Before (web/src/components/ConfigSection.tsx):
<h3>{section.label}</h3>
<p>{section.summary}</p>

// After:
import { useDescriptorT } from '../i18n/useDescriptorT';

const dt = useDescriptorT();
// …
<h3>{dt.sectionLabel(section.key, section.label)}</h3>
<p>{dt.sectionSummary(section.key, section.summary)}</p>
```

Same pattern for `field.label` / `field.help`, enum select options, and sidebar group names.

## 6. Language Toggle UI

### 6.1 Placement

TopBar right cluster, immediately left of the "Apply" button:

```
┌───────────────────────────────────────────────┐
│ hermind               [中|EN]  Apply  Discard │
└───────────────────────────────────────────────┘
```

### 6.2 Component (`web/src/components/shell/LanguageToggle.tsx`)

```tsx
import { useTranslation } from 'react-i18next';
import { setLanguage } from '../../i18n';
import styles from './LanguageToggle.module.css';

export default function LanguageToggle() {
  const { i18n, t } = useTranslation('ui');
  const isZh = i18n.language.toLowerCase().startsWith('zh');

  return (
    <div className={styles.toggle} role="group" aria-label={t('language.switcher')}>
      <button
        type="button"
        className={isZh ? styles.active : ''}
        onClick={() => setLanguage('zh-CN')}
        aria-pressed={isZh}
      >中</button>
      <button
        type="button"
        className={!isZh ? styles.active : ''}
        onClick={() => setLanguage('en')}
        aria-pressed={!isZh}
      >EN</button>
    </div>
  );
}
```

### 6.3 Switch Behavior

- `i18n.changeLanguage` fires — all components using `useTranslation` / `useDescriptorT` re-render automatically.
- `localStorage.setItem('hermind.lang', lang)` persists.
- `document.documentElement.lang` updates (for a11y + CJK font fallback).
- **Zero state loss**: dirty config edits, open dialogs, sidebar selection — all preserved. Switching language is not a navigation event.

## 7. Frontend String Inventory & Key Naming

### 7.1 Scan Result

`grep` scan (`>[A-Z][a-zA-Z ]{3,}<|label="[A-Z]…"|placeholder="…"|aria-label="…"`) across `web/src/**/*.tsx` yields **57 occurrences** in 20 files. Categorized:

| Category          | Examples                                         | Key pattern   |
| ----------------- | ------------------------------------------------ | ------------- |
| Top actions       | "Apply Changes", "Discard", "Reload"             | `action.*`    |
| Dialogs           | "New Provider", "Add MCP Server", "Cancel"       | `dialog.*`    |
| Empty / coming-soon | "Nothing configured yet", "Coming soon"        | `empty.*`    |
| Field controls    | "Show", "Hide", "Add", "Remove", "required"      | `field.*`    |
| Test connection   | "Test", "Testing...", "Connected", "Failed"      | `test.*`     |
| Status / errors   | "Unsaved changes", "Failed to load"              | `status.*`   |
| Footer / meta     | version string                                   | `footer.*`   |
| i18n meta         | "Language"                                       | `language.*` |

### 7.2 Naming Conventions

- **Dotted two-segment keys** (`category.name`), mirroring descriptor path style.
- **English base keys** — no Chinese / pinyin key names.
- **camelCase for name segment** (JSON convention).
- **Interpolation uses `{{var}}`** (i18next default).
- **Pluralization uses CLDR suffixes** (i18next v4 default): English uses `_one` + `_other`; Chinese uses `_other` only. Example: `status.unsavedChanges_one` = `"{{count}} unsaved change"`, `status.unsavedChanges_other` = `"{{count}} unsaved changes"`. Lookup via `t('status.unsavedChanges', { count })`.

### 7.3 `locales/en/ui.json` Seed

Flat top-level keys (same reason as §5.1 — `keySeparator: false`). Dot in the key is literal:

## 8. CI Completeness Gate

### 8.1 Problem

Developer adds a new field to `config/descriptor/browser.go` but forgets to add translations. Chinese UI silently displays English fallback — no crash but a visual fracture.

### 8.2 Two-Gate Solution

**Gate A — Go-side fixture generator** (`config/descriptor/fixture_gen_test.go`, build-tagged):

```go
//go:build fixture

package descriptor

import (
    "encoding/json"
    "os"
    "path/filepath"
    "testing"
)

// TestDumpSchemaFixture writes a canonical snapshot of the descriptor registry
// to web/src/i18n/__fixtures__/config-schema.json for the frontend
// completeness test to consume.
//
// Run: go test -tags fixture ./config/descriptor -run TestDumpSchemaFixture
func TestDumpSchemaFixture(t *testing.T) {
    out := filepath.Join("..", "..", "web", "src", "i18n", "__fixtures__", "config-schema.json")
    sections := All()
    data, err := json.MarshalIndent(sections, "", "  ")
    if err != nil {
        t.Fatal(err)
    }
    if err := os.MkdirAll(filepath.Dir(out), 0o755); err != nil {
        t.Fatal(err)
    }
    if err := os.WriteFile(out, append(data, '\n'), 0o644); err != nil {
        t.Fatal(err)
    }
}
```

Fixture is committed. CI invokes `go test -tags fixture ./config/descriptor -run TestDumpSchemaFixture` then `git diff --exit-code web/src/i18n/__fixtures__/config-schema.json`. Non-empty diff fails CI with: **"Run `go test -tags fixture ./config/descriptor -run TestDumpSchemaFixture` and commit the updated fixture."**

**Gate B — Frontend completeness test** (`web/src/i18n/completeness.test.ts`):

```typescript
import { describe, it, expect } from 'vitest';
import schema from './__fixtures__/config-schema.json';
import enDesc from '../locales/en/descriptors.json';
import zhDesc from '../locales/zh-CN/descriptors.json';

// Flat-key lookup (matches keySeparator: false in §4.1).
type FlatTree = Record<string, string>;

describe.each([
  ['en', enDesc as FlatTree],
  ['zh-CN', zhDesc as FlatTree],
])('descriptor translations — %s', (locale, tree) => {
  it('every section has label + summary', () => {
    for (const section of schema as Array<{ key: string }>) {
      expect(tree[`${section.key}.label`], `${locale} ${section.key}.label missing`).toBeDefined();
      expect(tree[`${section.key}.summary`], `${locale} ${section.key}.summary missing`).toBeDefined();
    }
  });

  it('every field has label (+help when source had it)', () => {
    for (const section of schema as Array<{
      key: string;
      fields?: Array<{ name: string; help?: string }>;
    }>) {
      for (const field of section.fields ?? []) {
        const labelKey = `${section.key}.fields.${field.name}.label`;
        expect(tree[labelKey], `${locale} ${labelKey} missing`).toBeDefined();
        if (field.help) {
          const helpKey = `${section.key}.fields.${field.name}.help`;
          expect(tree[helpKey], `${locale} ${helpKey} missing`).toBeDefined();
        }
      }
    }
  });

  it('every enum field has enum value translations (except empty-string)', () => {
    for (const section of schema as Array<{
      key: string;
      fields?: Array<{ name: string; kind: string; enum?: string[] }>;
    }>) {
      for (const field of section.fields ?? []) {
        if (field.kind !== 'enum') continue;
        for (const val of field.enum ?? []) {
          if (val === '') continue;
          const key = `${section.key}.fields.${field.name}.enum.${val}`;
          expect(tree[key], `${locale} ${key} missing`).toBeDefined();
        }
      }
    }
  });

  it('every GroupID referenced by sections has a groups.<id> entry', () => {
    const ids = new Set(
      (schema as Array<{ groupId?: string }>).map((s) => s.groupId).filter(Boolean) as string[],
    );
    for (const id of ids) {
      expect(tree[`groups.${id}`], `${locale} groups.${id} missing`).toBeDefined();
    }
  });
});
```

Either gate failing = CI red. Together they catch any divergence between backend descriptors and frontend translations.

## 9. Testing Strategy

| Layer | File | Covers |
| --- | --- | --- |
| Unit | `i18n/detect.test.ts` | `navigator.language='zh-CN'` → `'zh-CN'`; `='en-US'` → `'en'`; localStorage beats navigator; invalid stored value ignored |
| Unit | `i18n/useDescriptorT.test.tsx` | miss returns fallback; hit returns translation; language-switch triggers re-render |
| Unit | `i18n/completeness.test.ts` | §8.2 Gate B |
| Component | `components/shell/LanguageToggle.test.tsx` | click 中 → `i18n.language === 'zh-CN'`; click EN → `'en'`; `aria-pressed` updates; localStorage written |
| Component | `App.test.tsx` (updated) | initial render uses current language; language switch re-renders all ui strings |
| Integration | Go `TestDumpSchemaFixture` (build tag) | fixture matches current descriptor registry |

**Test gotchas:**
- `vitest` default env `jsdom` provides `navigator.language` and `localStorage`. 
- `initI18n()` has global side effects. Reset per-test: add `beforeEach(() => i18n.changeLanguage('en'))` in `web/src/test/setup.ts`.
- Mock `localStorage` getter when testing detection priority (`vi.spyOn(Storage.prototype, 'getItem')`).

## 11. Risks & Mitigations

| Risk | Mitigation |
| --- | --- |
| Inconsistent / clunky Chinese translation | Project owner authors v1; feedback loop via GitHub issues tagged `i18n` |
| Developer adds field, forgets translation | §8 two-gate CI blocks merge |
| Translation bundle grows unbounded | 2 × ~200 keys ≈ 10 kB; beyond 5 locales, revisit with dynamic import |
| `initI18n()` rejects, blocks render | `main.tsx` uses `.finally()` — UI mounts regardless; console error surfaces failure |
| Untranslated JSX literals slip through | Manual PR review; optional future lint rule (not in this PR) |
| Locale-specific font rendering flaws | `<html lang>` sync lets browsers pick the right CJK fallback automatically |
