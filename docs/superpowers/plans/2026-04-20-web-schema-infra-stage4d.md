# Web Config Schema Infrastructure (Stage 4d) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cross-section datalist — make the Default Model scalar field autocomplete from every `providers.*.model` value the user has already configured. Type ahead, pick from the list. Remove the guesswork of "wait, what did I call that Anthropic model again?".

**Architecture:** The schema descriptor already routes field rendering through `ConfigSection`. Stage 4d adds an optional `DatalistSource` hint on `FieldSpec` — a pointer to another section's field whose values populate a native HTML `<datalist>`. The emission path: Go `FieldSpec.DatalistSource` → API DTO `datalist_source` → Zod schema → React `TextInput` wires a `list=` attribute and renders a sibling `<datalist>` populated from `state.config[source.section]`. Strictly opt-in per field — existing text inputs render unchanged.

**Tech Stack:** Go 1.21+ (config/descriptor, api), TypeScript + React + Zod (web/src), Vitest, Go `testing`, react-testing-library.

**Spec:** Reuse `docs/superpowers/specs/2026-04-20-web-schema-infra-stage4b-design.md` §"Open questions carried into 4c" — this ships the "cross-section datalist" open question from that list. No separate 4d design doc.

**Scope note — what this does NOT ship:**
- **Only the Default Model field.** Per-provider `model` fields (inside ProviderEditor and FallbackProviderEditor) keep rendering as plain text inputs. Those already have their own provider-specific datalist via fetch-models (Stage 4b), and wiring cross-section for them adds noise without clear user value.
- **Only one source section per field.** `DatalistSource` is a single pointer, not an array. If the user types a new model in fallback-only, it will not appear in Default Model's autocomplete. Acceptable — configuring a fallback without a matching primary provider is rare, and we can promote to multi-source in a later stage if real users complain.
- **No server-side collection.** The datalist values are computed on the client from `state.config`, not fetched from a new endpoint. Zero new network calls.
- **No type coercion beyond string.** `DatalistSource` assumes string values. Pointing at a non-string field silently yields no suggestions.
- **No invariant test that the source section/field exists.** Descriptor authors are responsible for pointing at real targets; a typo produces an empty datalist, not a test failure. Could be added as a tripwire later.

```tsx
import { useId } from 'react';
import styles from './fields.module.css';
import type { SchemaField } from '../../api/schemas';

export interface FieldProps {
  field: SchemaField;
  value: string;
  onChange: (value: string) => void;
  /** Optional autocomplete suggestions. When non-empty, a sibling <datalist>
   *  renders and the input's list= attribute wires up to it. */
  datalist?: readonly string[];
}

export default function TextInput({ field, value, onChange, datalist }: FieldProps) {
  const listId = useId();
  const hasList = Array.isArray(datalist) && datalist.length > 0;
  return (
    <label className={styles.row}>
      <span className={styles.label}>
        {field.label}
        {field.required && <span className={styles.required}>*</span>}
      </span>
      <input
        type="text"
        className={styles.input}
        value={value}
        placeholder={field.default !== undefined ? String(field.default) : undefined}
        onChange={e => onChange(e.currentTarget.value)}
        list={hasList ? listId : undefined}
      />
      {hasList && (
        <datalist id={listId}>
          {datalist!.map(v => (
            <option key={v} value={v} />
          ))}
        </datalist>
      )}
      {field.help && <span className={styles.help}>{field.help}</span>}
    </label>
  );
}
```

```tsx
function collectDatalistValues(
  config: Record<string, unknown> | undefined,
  sectionKey: string,
  fieldName: string,
): string[] {
  if (!config) return [];
  const blob = (config as Record<string, unknown>)[sectionKey];
  const out = new Set<string>();
  if (Array.isArray(blob)) {
    for (const el of blob) {
      if (el && typeof el === 'object') {
        const v = (el as Record<string, unknown>)[fieldName];
        if (typeof v === 'string' && v !== '') out.add(v);
      }
    }
  } else if (blob && typeof blob === 'object') {
    for (const el of Object.values(blob as Record<string, unknown>)) {
      if (el && typeof el === 'object') {
        const v = (el as Record<string, unknown>)[fieldName];
        if (typeof v === 'string' && v !== '') out.add(v);
      }
    }
  }
  return Array.from(out).sort();
}
```
